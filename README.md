# 孤军：四行1937

一款以四行仓库保卫战为背景的文字策略网页游戏。2.5 版采用“本地规则引擎 + 可选 AI 文字增强”架构：所有兵力、资源、战斗、事件与结局都在浏览器内即时结算，断网或 AI 服务不可用时仍能完整游玩。

## 2.5 更新

- 战略地图压缩为四张迷你楼层卡，手机打开后最多占约三分之一屏幕。
- 默认只显示驻军、完整度、楼层状态和敌军目标，工事、机枪与操作集中到选中楼层详情。
- 新增动态战线：红色敌军标记沿主攻线移动，速度随敌袭威胁变化，被锁定楼层同步亮起警示。
- 敌军路线会随楼层失守或夺回即时改变，但游戏仍采用回合制，不会在玩家未操作时自动结算。
- 顶部状态名称进一步缩短，并移除重复的核心火力说明栏。

## 2.4 更新

- 重做顶部战况总览，以时间、指挥部、核心指标、敌情威胁和资源五个清晰区域取代密集小字仪表盘。
- 核心数字提升至 14–20px，重要说明不低于 11–12px，并增强文字与背景对比度。
- 重做战略地图卡片、选中防区详情、指挥官风险和失守后果提示，减少重复信息。
- 放大快捷命令、行动预览、战报结算、存档和战地顾问文字，改善手机触控面积与横向浏览。
- 移除滚动情报跑马灯，避免动画和多色小字争夺注意力。

## 2.3 更新

- 四个防区拥有独立完整度；一楼失守后敌军会向二楼或地下室推进，防区变化不再只是视觉效果。
- 防区失守时驻军自动后撤，未提前转移的机枪组可能成功撤离、弃枪转为步兵或被敌军截断。
- 地下室医院失守会立即损失伤员、药品和弹药，此后伤员恶化时间减半且无法接受成体系治疗。
- 突破口出现后可以封锁通往下一层的楼梯，一次性降低敌军进攻规模；也可以消耗兵力与弹药反冲锋夺回失地。
- 指挥官所在楼层遭袭时，以 2% 为中等防御基准判定阵亡；工事、驻军、机枪和防区完整度可将概率动态调整至约 0.3%–5%。
- 新增“将星陨落”独立结局、动态进攻路线、地图连锁后果说明与对应本地战地顾问规则。

## 2.2 更新

- 重做仓库剖面战略地图：屋顶、二楼、一楼和地下室分别显示驻军、工事、重机枪与战术作用。
- 可以调派 30 名士兵或移动单个机枪班；敌军只与受攻击区域的部署交战，布防会真实改变伤亡和阵地损失。
- 时间统一为分钟制，正确处理午夜、跨日和十月至十一月的日期，不再因结算顺序覆盖战报。
- “兵力”改为与失败判定一致的“可战兵力”，首次濒临崩溃会触发一次最后防线警告，不再在尚有战力时莫名结束。
- 结局报告显示准确原因、日期与可战兵力；关闭结局后可以返回战场查看带时间戳的完整操作记录，并可随时重新打开报告。

## 2.1 更新

- 行动前显示耗时、威胁变化、资源消耗与风险等级。
- 战斗后生成独立结算卡，清楚列出兵力、伤员、士气、阵地和弹药变化。
- 六天战况各有独立主题、敌情和进攻倾向，后期炮击与总攻压力会明显升级。
- 每次命令结算后自动保存，并提供 5 个手动档案位置；旧版存档会自动迁移。
- 增加明确的“跳过”打字按钮、本地音效开关和手机端紧凑布局。
- GitHub Pages 自动识别为纯静态环境，直接使用本地叙事与本地顾问，不会请求不存在的 API。

## 直接运行

需要 Node.js 20 或更高版本。

```bash
npm install
npm run dev
```

浏览器打开终端显示的本地地址。生产构建与完整检查：

```bash
npm run check
npm run preview
```

`dist/` 是可直接部署的静态网页版本；首次在线打开后，浏览器会缓存已访问的游戏文件用于离线启动。

## GitHub Pages 部署

仓库已经包含 `.github/workflows/deploy-pages.yml`。将源码推送到公开 GitHub 仓库的 `main` 分支后，在仓库的 **Settings → Pages → Build and deployment** 中把 **Source** 设为 **GitHub Actions**，随后工作流会自动测试、构建并发布 `dist/`。

游戏使用相对资源路径，可在 `https://<用户名>.github.io/<仓库名>/` 这样的项目子路径中运行。GitHub Pages 不保存服务端密钥，因此该部署默认使用完整的本地游戏引擎与本地战地顾问。

## 免费 AI 增强（可选）

可选增强默认接入硅基流动的 OpenAI 兼容接口，模型为 `Qwen/Qwen2.5-7B-Instruct`。游戏内 AI 开关默认关闭；没有配置密钥时，即使打开也会自动回退到本地叙事和本地顾问，不会卡住游戏。

1. 复制 `.env.example` 为 `.env.local`。
2. 只填写服务端变量 `SILICONFLOW_API_KEY`，不要使用 `VITE_` 前缀。
3. 重启开发服务器。

密钥只由 Vite 本地服务或 Netlify Function 读取，不会打包到浏览器 JavaScript。生产部署时，请在托管平台的环境变量设置中填写密钥，切勿把真实密钥写进源码。

Netlify 已包含 `netlify.toml` 与 `netlify/functions/narrate.mjs`。其他静态托管平台即使没有部署 `/api/narrate`，本地游戏仍可完整运行，只是没有 AI 润色。

## 代码结构

- `engine/`：本地游戏规则、行动风险预览、回合结算、命令意图与结局判断。
- `data/dayProfiles.ts`：六天战况主题与敌军压力参数。
- `data/text/`：本地战报、事件和对白模板。
- `services/aiClient.ts`：可取消、限时的可选 AI 客户端；失败即回退本地内容。
- `server/`、`netlify/functions/`：服务端 AI 网关，负责保护密钥。
- `storage/`：自动存档、5 个手动槽位、版本化存档与旧存档迁移。
- `public/`：PWA 清单、图标与离线缓存脚本。
- `tests/`：意图识别、结局和存档迁移测试。

## 设计约束

- AI 只能润色文字，不能修改数值、胜负、事件或存档。
- 撤退命令必须精确匹配并二次确认；移动命令不会误触结局。
- 随机数状态随存档保存，同一存档可稳定复现后续随机结果。
- 外部纹理、Google 字体、Tailwind CDN 和 Google AI SDK 均已移除。
- AI 请求最长等待 8 秒，可手动取消，本地战报始终先显示。
